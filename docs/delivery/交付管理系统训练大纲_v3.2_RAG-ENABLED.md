## 🏗️ RAG私有知识库标准架构

### 统一技术栈
- **向量数据库**: SQLite + FAISS 轻量化实现（符合"内置轻量数据库"原则）
- **嵌入模型**: 本地部署或标准API
- **检索策略**: 混合检索（关键词 + 向量相似度）
- **分块策略**: 语义分块 + 重叠窗口

### 统一数据模型
```
data/delivery_management.db 中新增RAG相关表：
├── rag_collections              # 知识库集合表
│   ├── id
│   ├── name (product/delivery/cissp)
│   ├── description
│   ├── created_at
│   └── updated_at
├── rag_documents                # 文档表
│   ├── id
│   ├── collection_id
│   ├── title
│   ├── source_file
│   ├── content_hash
│   ├── created_at
│   └── updated_at
├── rag_chunks                   # 文本块表
│   ├── id
│   ├── document_id
│   ├── content
│   ├── chunk_index
│   ├── token_count
│   └── metadata_json
└── rag_vectors                  # 向量表（FAISS索引文件单独存储）
    ├── chunk_id
    └── vector_blob
```

### 统一API接口（Iris 负责）

```bash
# 1. 知识库管理
python -m skills.rag_kb create-collection --name "产品服务知识库"
python -m skills.rag_kb list-collections
python -m skills.rag_kb delete-collection --name "产品服务知识库"

# 2. 文档入库
python -m skills.rag_kb add-document --collection "product" --file "产品说明.docx"
python -m skills.rag_kb batch-import --collection "product" --folder "./data/products"

# 3. 知识库查询（默认纯检索，不调用大模型）
python -m skills.rag_kb search --collection "product" --query "移动应用加固有哪些方案"

# 4. RAG增强查询（显式开启才调用大模型）
python -m skills.rag_kb search --collection "product" --query "移动应用加固有哪些方案" --enable-llm
```

## 🎯 RAG架构的核心优势

### 1. ✅ 符合"脚本为主，大模型仅做增强"原则
- **默认模式（纯检索）**：不调用大模型，纯向量+关键词检索，离线可用
- **增强模式（RAG问答）**：显式开启 `--enable-llm` 才调用大模型

### 2. ✅ 符合"内置轻量数据库"原则
- SQLite存储元数据 + FAISS本地文件存储向量
- 零外部依赖，单文件可部署

### 3. ✅ 充分利用4月已有成果
- 3个知识库数据已存在，只需导入RAG引擎建立索引
- 不是从零构建知识库，而是升级为RAG架构
