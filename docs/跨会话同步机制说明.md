# 跨会话同步机制使用说明

## 概述

本机制解决了 **WebChat**、**企业微信** 等多会话通道的上下文隔离问题，确保：
- ✅ 所有通道共享同一套记忆
- ✅ 训练进度、业务场景状态全局同步
- ✅ 用户的重要指令在所有通道都能看到

---

## 机制架构

```
WebChat 通道 ───┐
                  ├──► 共享记忆层 ◄──┐
企业微信通道 ────┘                      │
                                        ▼
                                  MEMORY.md (长期记忆)
                                  memory/.sync_state.json (同步状态)
                                  memory/.cross_session_sync.json (配置)
                                  memory/YYYY-MM-DD.md (日报)
```

---

## 核心文件

| 文件 | 作用 |
|------|------|
| `MEMORY.md` | 长期记忆的核心，所有会话优先读取 |
| `memory/.sync_state.json` | 同步状态：最近更新、重要提示、同步历史 |
| `memory/.cross_session_sync.json` | 同步配置：5个业务场景全局状态 |
| `scripts/sync_session.py` | 同步检查脚本 |

---

## 自动同步规则

### 🔄 触发自动同步的场景

1. **完成训练任务** → 自动更新业务场景状态
2. **产生重要决策** → 写入 `important_notes`，所有通道可见
3. **新增/修改业务场景** → 立即更新全局状态
4. **用户配置变更** → 同步到所有通道
5. **待办事项变化** → 全局同步

### 📋 同步优先级

| 优先级 | 内容 | 同步方式 |
|--------|------|---------|
| 🔴 高 | 用户重要指令、系统配置变更 | 立即同步 |
| 🟡 中 | 训练任务完成、业务场景更新 | 任务完成时同步 |
| 🟢 低 | 一般性对话、常规操作 | 不主动同步 |

## 使用方法

### 1. 新会话启动时检查同步状态

```bash
python3 scripts/sync_session.py <channel>
```

通道名称示例：`wecom`、`webchat`、`discord`

### 2. 记录同步事件

```bash
python3 scripts/sync_session.py <channel> record "事件描述" [变更1] [变更2]...
```

### 3. 添加重要提示（所有通道可见）

```python
from sync_session import add_important_note

add_important_note('wecom', '用户要求删除所有原始文档', priority='high')
```

### 4. 更新业务场景状态

```python
from sync_session import update_business_scenario

update_business_scenario('scenario_5', 'completed', {'comment': 'ONES筛选器扩展到10个'})
```

## 常见问题

### Q: 为什么在 WebChat 说的话，企业微信看不到？
A: **OpenClaw 的设计是会话隔离的**，但本机制通过以下方式解决：
1. **共享记忆层**：所有重要内容都写入共享文件
2. **启动检查**：每次会话启动都检查 `.sync_state.json`
3. **重要提示**：`important_notes` 在所有通道都会显示

### Q: 同步是否实时？
A: 重要事件（训练完成、重要决策）是**立即同步**到共享文件，其他会话**下一次启动**时就能看到。

### Q: 如何确保不会遗漏重要信息？
A: 
1. 每次会话启动都会显示最近的 `important_notes`
2. 5个业务场景的全局状态始终显示
3. 最近同步事件和变更历史完整记录
