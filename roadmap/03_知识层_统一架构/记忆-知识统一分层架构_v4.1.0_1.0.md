# 🧠 记忆-知识统一分层架构 v4.0.0

> **版本**：v4.0.0 统一架构版
> **日期**：2026-05-09
> **核心理念**：记忆是流动的知识，知识是沉淀的记忆
> **架构原则**：分层存储、自动流动、价值筛选、永久沉淀

---

## 🎯 问题：过去的架构缺陷

```
❌ 问题 1：记忆与知识库割裂
   memory/ 目录与 knowledge-base/ 各自独立，数据不流动

❌ 问题 2：缺少价值筛选机制
   所有内容不分价值全部保存，噪音越来越多

❌ 问题 3：没有明确的分层边界
   短期对话、中期经验、长期知识混在一起，检索效率低

❌ 问题 4：无法组织化学习
   一个智能体学会的东西，其他智能体无法复用
```

---

## 🏗️ 统一四层架构设计

```
┌─────────────────────────────────────────────────────────────────┐
│  L1 瞬时记忆 (Working Memory)                                      │
│  范围：当前会话 / 上下文窗口                                      │
│  保存时间：分钟级 → 会话结束即丢弃                                 │
│  存储方式：内存 / 会话上下文                                      │
│  数据量：KB 级                                                    │
│  典型内容：当前对话、用户问题、中间思考过程                       │
└──────────────────────────────┬──────────────────────────────────┘
                               │ ✅ 有价值 → 自动下沉
┌──────────────────────────────▼──────────────────────────────────┐
│  L2 短期记忆 (Short-term Memory)                                 │
│  范围：当日 / 近 7 天                                           │
│  保存时间：天级 → 7 天后自动评估                                 │
│  存储方式：memory/YYYY-MM-DD.md (文件系统)                      │
│  数据量：MB 级                                                    │
│  典型内容：当日对话记录、操作日志、临时决策                     │
└──────────────────────────────┬──────────────────────────────────┘
                               │ ✅ 达标 → 自动沉淀
┌──────────────────────────────▼──────────────────────────────────┐
│  L3 中期记忆 (Mid-term Memory)                                   │
│  范围：7 天 - 1 年                                               │
│  保存时间：年级 → 年度评估                                       │
│  存储方式：SQLite 知识库 (结构化)                               │
│  数据量：GB 级                                                    │
│  典型内容：错误案例、最佳实践、决策记录、领域知识             │
└──────────────────────────────┬──────────────────────────────────┘
                               │ ✅ 精华 → 永久晋升
┌──────────────────────────────▼──────────────────────────────────┐
│  L4 长期知识 (Long-term Knowledge)                                │
│  范围：永久保存                                                  │
│  保存时间：永久 → 作为智能体的 "基因"                           │
│  存储方式：进化模式库 / 核心规则文件 / 角色定义                 │
│  数据量：MB 级（精选精华）                                       │
│  典型内容：安全闸门规则、错误模式库、最佳实践范式、组织文化   │
└─────────────────────────────────────────────────────────────────┘
```

## 🔄 记忆→知识 自动流动管道

### 流动规则：自动筛选 + 人工审核

```
L1 瞬时记忆
    ↓ 会话结束触发
【记忆价值评估器 v1.0】
    ├─ 评估维度：
    │   ├─ 创新性 ← 是否包含新的知识/方法
    │   ├─ 复用性 ← 是否可能被其他场景使用
    │   ├─ 正确性 ← 是否经过验证正确有效
    │   ├─ 风险性 ← 是否包含错误/教训/踩坑
    │   └─ 敏感度 ← 是否包含敏感信息（需要脱敏）
    │
    ├─ 评分算法：5 维度各 0-20 分，总分 ≥ 60 分 → 流动
    │
    └─ 自动操作：
        ├─ < 60 分 → 丢弃（不保存）
        └─ ≥ 60 分 → 写入 L2 短期记忆

    ↓ 每日 02:00 定时任务
【每日记忆蒸馏器】
    ├─ 提取当日所有 ≥ 60 分的对话
    ├─ 去除重复、冗余、噪音内容
    ├─ 自动分类打标签（错误/最佳实践/决策/知识）
    ├─ 生成摘要 + 关键词
    │
    └─ 自动操作：
        ├─ 普通内容 → 保留在 L2
        └─ 高分内容（≥ 80 分）→ 推荐进入 L3 + 通知人工审核

    ↓ 每周日 03:00 定时任务
【每周知识提炼器】
    ├─ 回顾本周所有 L2 记忆
    ├─ 识别重复出现的模式
    ├─ 关联相似知识，发现关联关系
    ├─ 评估实际使用频率（被检索次数）
    │
    └─ 自动操作：
        ├─ 高频使用 → 自动晋升 L3 中期记忆
        ├─ 低频使用 → 降低优先级 / 归档
        └─ 从未使用 → 自动删除

    ↓ 每年 12 月 年度任务
【年度基因进化器】
    ├─ 回顾全年所有 L3 知识
    ├─ 提炼最高频、最高价值的模式
    ├─ 抽象为可复用的规则/方法论
    │
    └─ 自动操作：
        ├─ 写入 L4 核心基因库
        ├─ 更新安全闸门规则
        ├─ 更新最佳实践库
        └─ 更新错误模式库
```

## 📊 各层详细设计

### L1 瞬时记忆：会话级上下文

```
存储位置：内存 / 会话对象
生命周期：会话期间（分钟 - 小时）

设计要点：
✅ 不持久化，会话结束即清理
✅ 支持上下文窗口滑动
✅ 关键决策点可标记为「待持久化」
✅ Token 数严格控制，避免超出模型限制

典型内容：
├─ 用户当前问题
├─ 中间思考步骤
├─ 工具调用结果
└─ 已生成的回答片段
```

---

### L2 短期记忆：每日记忆库

```
存储位置：memory/YYYY-MM-DD.md
生命周期：7 天（可配置）

设计要点：
✅ 按日期组织，每天一个文件
✅ 原始对话记录，完整保存
✅ 自动打标签（按领域/类型/价值评分）
✅ 7 天后自动触发二级评估

文件结构：
---
date: 2026-05-09
overall_score: 72
high_value_count: 3
---

## [10:30] 知识库架构设计讨论 [价值: 85分]
标签: 架构设计, 知识库, 记忆系统

用户：Jerry，知识库和记忆系统是否应该整合？
Jerry：分析了当前架构的 4 个主要问题...
...

## [14:20] 合同审查任务 [价值: 68分]
...

自动评估字段：
├─ value_score: 0-100 (整体价值评分)
├─ has_error: true/false (是否包含错误案例)
├─ has_best_practice: true/false (是否包含最佳实践)
├─ domain: 标签分类
└─ keywords: 关键词列表
```

### L3 中期记忆（知识库 v4.0）：组织级知识资产

```
存储位置：knowledge-base/knowledge.db (SQLite)
生命周期：7 天 - 1 年

这就是我们刚刚建设的知识库系统！

核心表：
├─ knowledge_sources → 数据源（包含 agent_memory 类型）
├─ documents → 文档主表（从 L2 晋升的记忆）
├─ document_chunks → 分块内容
├─ chunk_vectors → 向量索引
├─ knowledge_tags → 标签体系
├─ knowledge_relations → 知识关联关系
└─ query_logs → 查询使用记录（用于评估知识价值）

自动晋升触发条件（满足任一即可）：
✅ 价值评分 ≥ 80 分
✅ 被检索并使用 ≥ 3 次
✅ 被标记为「错误案例」或「最佳实践」
✅ 包含新的方法论或流程改进
```

---

### L4 长期知识（基因库）：智能体的核心 DNA

```
存储位置：roadmap/ 目录下的核心规则文件
生命周期：永久（智能体的基因）

核心文件：
├── roadmap/
│   ├── SOUL.md          # 使命、愿景、价值观（最高层）
│   ├── AGENT.md         # 行为规则、安全闸门、决策原则
│   ├── 进化模式库.json   # 抽象出的可复用模式（错误模式/最佳实践模式）
│   ├── 错误模式库.json   # 经过验证的错误预防模式
│   └── 角色定义/        # 各 Agent 的角色定位与能力边界

晋升机制（极为严格，宁缺毋滥）：
✅ 在 L3 中存在超过 6 个月
✅ 被引用使用 ≥ 20 次
✅ 经过 3 次以上人工审核确认
✅ 证明能有效预防错误或提升效率
✅ 人工最终确认「这应该成为我们的基因」

基因更新频率：
├─ 安全规则 → 发现即更新（最高优先级）
├─ 错误模式 → 月度评审
├─ 最佳实践 → 季度评审
└─ 核心价值观 → 年度评审（极为慎重）
```

## 🛠️ 核心组件设计

### 1. 记忆价值评估器 (MemoryValueAssessor)

```python
class MemoryValueAssessor:
    """评估对话记忆的价值，决定是否持久化"""
    
    def assess(self, conversation: List[Message]) -> Dict:
        return {
            "value_score": self._calc_value_score(conversation),
            "has_error": self._detect_error_case(conversation),
            "has_best_practice": self._detect_best_practice(conversation),
            "domain_tags": self._auto_tagging(conversation),
            "keywords": self._extract_keywords(conversation),
            "should_persist": score >= 60,
            "should_recommend_l3": score >= 80
        }
    
    def _calc_value_score(self, conversation: List[Message]) -> int:
        """5 维度加权评分"""
        scores = {
            "innovation": self._innovation_score(conversation),   # 创新性 0-20
            "reusability": self._reusability_score(conversation), # 复用性 0-20
            "correctness": self._correctness_score(conversation), # 正确性 0-20
            "risk_value": self._risk_value_score(conversation),   # 风险价值 0-20
            "generality": self._generality_score(conversation)    # 普适性 0-20
        }
        return sum(scores.values())
```

### 2. 每日记忆蒸馏器 (DailyMemoryDistiller)

```python
class DailyMemoryDistiller:
    """每日定时任务，蒸馏当日高价值记忆"""
    
    def run(self, date: str):
        # 1. 读取当日所有 L2 记忆
        memories = self._load_daily_memory(date)
        
        # 2. 去重、去噪
        cleaned = self._clean_redundancy(memories)
        
        # 3. 分类聚类
        clusters = self._cluster_by_topic(cleaned)
        
        # 4. 生成摘要
        summaries = self._generate_summaries(clusters)
        
        # 5. 推荐晋升 L3
        for item in cleaned:
            if item["value_score"] >= 80:
                self._recommend_promotion_to_l3(item)
        
        # 6. 写入当日蒸馏报告
        self._write_report(summaries)
```

### 3. 知识生命周期管理器 (KnowledgeLifecycleManager)

```python
class KnowledgeLifecycleManager:
    """管理知识从 L2 → L3 → L4 的完整生命周期"""
    
    def check_promotion_candidates(self):
        """检查符合晋升条件的知识"""
        return kb.execute_query("""
            SELECT d.*, COUNT(q.id) as usage_count
            FROM documents d
            LEFT JOIN query_logs q ON d.id = q.document_id
            WHERE d.created_at > DATE('now', '-7 days')
            GROUP BY d.id
            HAVING usage_count >= 3
        """)
    
    def auto_promote(self):
        """自动晋升符合条件的知识"""
        for candidate in self.check_promotion_candidates():
            if self._human_approval_required(candidate):
                self._notify_for_review(candidate)
            else:
                self._do_promote(candidate)
```

## 📊 各层边界清晰定义

| 层级 | 名称 | 存储 | 生命周期 | 检索范围 | 访问频率 |
|------|------|------|---------|---------|---------|
| **L1** | 瞬时记忆 | 内存 | 分钟/小时 | 当前会话 | 极高 |
| **L2** | 短期记忆 | 文件系统 | 天级 (7天) | 当日/近周 | 高 |
| **L3** | 中期记忆 | SQLite 知识库 | 月/年级 | 全量检索 | 中 |
| **L4** | 长期基因 | 核心规则文件 | 永久 | 启动时加载 | 低（但影响全局） |

## 🎯 整合后的核心优势

| 优势 | 说明 |
|------|------|
| **自动流动** | 记忆从产生到沉淀完全自动化，无需人工干预 |
| **价值筛选** | 垃圾信息及时丢弃，高价值知识自动沉淀 |
| **分层检索** | 优先检索高层知识，效率大幅提升 |
| **组织学习** | 一个智能体的经验，全组织智能体共享 |
| **持续进化** | 使用频率越高的知识，优先级越高 |
| **基因稳定** | L4 核心基因变化极慢，保证系统稳定性 |
| **永不遗忘** | 真正有价值的知识永久保存 |

## 💡 关键设计哲学

> **「记忆是流动的知识，知识是沉淀的记忆」**
>
> - 不是所有记忆都值得成为知识
> - 不是所有知识都值得成为基因
> - 价值是流动的唯一标准
> - 使用频率是最好的价值评估器
> - 时间是最好的过滤器
