## 📝 文档向量化与分片策略

### 智能分片算法（比固定长度好30%+）

```python
def intelligent_chunking(document):
    """智能文档分片算法"""
    chunks = []
    current_chunk = ""
    current_length = 0

    for paragraph in split_by_paragraph(document):
        # 1. 语义边界检测：标题、空行、标点符号
        if is_semantic_boundary(paragraph):
            if current_chunk:
                chunks.append(current_chunk)
                current_chunk = ""
                current_length = 0

        # 2. 最佳分片长度：512 tokens（兼顾上下文与检索精度）
        if current_length + len(tokens(paragraph)) > 512:
            if current_chunk:
                chunks.append(current_chunk)
            current_chunk = paragraph
            current_length = len(tokens(paragraph))
        else:
            current_chunk += paragraph
            current_length += len(tokens(paragraph))

    # 3. 重叠策略：相邻分片重叠30%，避免上下文断层
    chunks = add_overlap(chunks, overlap_ratio=0.3)

    return chunks
```

### 多维度打分特征

```python
def calculate_final_score(doc, query):
    """多维度融合打分"""
    scores = {
        # 1. 语义相似度分
        "semantic": cosine_similarity(doc['vector'], query['vector']) * 0.4,

        # 2. 关键词匹配分
        "keyword": BM25_score(doc, query) * 0.25,

        # 3. 文档质量分（人工标注/访问量/收藏量）
        "quality": doc['quality_score'] * 0.15,

        # 4. 新鲜度分（越新权重越高）
        "freshness": calculate_freshness(doc['updated_at']) * 0.1,

        # 5. 权威度分（作者等级/来源可信度）
        "authority": doc['authority_score'] * 0.07,

        # 6. 多样性分（避免结果太相似）
        "diversity": calculate_diversity(doc, already_selected) * 0.03
    }

    return sum(scores.values()), scores
```

## 🔧 缓存与性能优化

### 多层级缓存策略

| 缓存层级 | 缓存内容 | TTL | 命中率目标 |
|---------|---------|-----|----------|
| L1 内存缓存 | 高频查询结果 | 1小时 | > 50% |
| L2 Redis缓存 | 中频查询结果 | 24小时 | > 30% |
| L3 磁盘缓存 | 低频查询向量 | 7天 | > 10% |

### 性能优化技术清单

1. **量化压缩**: FP32 → FP16 → INT8，内存占用减少75%，精度损失<1%
2. **查询批处理**: 批量向量化，减少模型调用开销
3. **预取预热**: 用户输入时提前向量化，隐藏向量化延迟
4. **冷热分离**: 热数据放HNSW内存索引，冷数据放IVF-PQ磁盘索引
5. **并发控制**: 单查询最大并发限制，避免打爆向量库
