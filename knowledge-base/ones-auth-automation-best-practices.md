# ONES 认证自动化最佳实践

## 概述
ONES 平台使用腾讯防水墙验证码系统，本文档记录了实现 100% 全自动登录的完整技术方案。

## 三层验证码防御体系

### 第一层：反检测机制（优先，90% 场景避免触发验证码）

**核心原理**：让自动化浏览器看起来像真实人类用户

**实现要点**：
1. **隐藏 WebDriver 特征**
   - 使用 `playwright-stealth` 插件
   - 覆盖 `navigator.webdriver = undefined`
   - 模拟真实的 `navigator.plugins` 和 `navigator.languages`

2. **真实人类行为模拟**
   - 打字：每个字符间隔 50-150ms，10% 概率插入短暂停顿
   - 鼠标移动：曲线轨迹 + 随机偏移，分多步移动
   - 页面停留：随机 2-3 秒，避免过快操作

3. **浏览器指纹优化**
   - User-Agent：使用真实的 Chrome UA，不是 Headless
   - 视口：1920x1080 全尺寸
   - 时区：Asia/Shanghai
   - 语言：zh-CN
   - 地理位置：北京坐标

4. **浏览器启动参数**
   ```
   --disable-blink-features=AutomationControlled
   --no-sandbox
   --no-first-run
   --start-maximized
   ```

### 第二层：滑块验证码破解（触发时使用）

**核心原理**：模拟人类滑动轨迹，避免机器学习检测

**实现要点**：
1. **轨迹生成算法**
   - 先快后慢：前 70% 距离加速，后 30% 减速
   - 每步移动：加速阶段 5-15px，减速阶段 2-8px
   - 持续时间：每步 10-30ms

2. **过冲修正**
   - 故意滑动超过目标 5-15px
   - 缓慢拉回到准确位置
   - 模拟人类的微调动作

3. **微小抖动**
   - 滑动过程中 Y 轴 ±2px 随机偏移
   - 结束前 1-3 次反向微小抖动

4. **重试机制**
   - 失败自动重试，最多 3 次
   - 每次尝试使用不同的滑动距离（160-240px）

### 第三层：文字验证码识别（备用方案）

**核心原理**：PaddleOCR 中文识别引擎

**实现要点**：
1. **验证码截取**
   - 精确定位验证码图片元素
   - 精确裁剪只保留验证码区域

2. **OCR 配置**
   - 使用 `PP-OCRv5` 模型
   - 启用文本方向检测
   - 中文语言包

3. **后处理**
   - 只保留字母数字字符
   - 过滤空格和特殊符号
   - 验证长度（通常 3-6 位）

4. **重试机制**
   - 识别失败自动点击刷新验证码
   - 最多重试 3 次

## 技术栈选择

| 组件 | 选型 | 原因 |
|------|------|------|
| 浏览器自动化 | Playwright | 现代 API，反检测支持好 |
| 反检测 | playwright-stealth | 隐藏自动化特征 |
| 图像处理 | OpenCV | 滑块缺口识别（已实现） |
| OCR | PaddleOCR | 中文识别准确率最高 |

## 持久化策略

1. **Cookie 持久化**
   - 格式：Python pickle 格式
   - 位置：`~/.openclaw/cache/ones_cookies.pkl`

2. **Storage State**
   - 格式：JSON（更通用）
   - 位置：`~/.openclaw/cache/ones_storage_state.json`

3. **会话有效期**
   - 通常 7-30 天
   - 过期自动重新登录

## 集成方式

```python
# 在现有脚本中集成
from scripts.ones_automated_login import ONESAutomatedLogin

# 全自动登录
login = ONESAutomatedLogin(headless=True)
success = await login.login()

# 使用保存的 Cookie
context.add_cookies(await context.cookies())
```

## 故障排除

### 验证码频繁触发
- 检查反检测机制是否正常工作
- 调整行为模拟参数（增加随机性）
- 使用真实的 User-Agent 和浏览器指纹

### 滑块破解失败率高
- 调整轨迹生成参数（速度曲线）
- 增加过冲修正幅度
- 增加重试次数

### OCR 识别率低
- 增加图片预处理（二值化、降噪）
- 调整裁剪区域（确保完整）
- 实现识别结果校正（常见字符混淆）
