## 🎯 核心配置

### 两大核心原则

```json
{
  "skills": {
    "scriptFirst": true,    // 脚本优先，脚本能实现的绝对不用大模型
    "preferBuiltin": true    // 优先使用内置能力
  },
  "llmOptimization": {
    "minimizeCalls": true,  // 最小化大模型调用
    "cacheEnabled": true,     // 启用缓存
    "cacheTTL": 3600          // 1小时缓存
  }
}
```

### 3层模型降级链（通用）

```
主模型 → 备选模型1 → 备选模型2

示例：GPT-4 → Claude-3 → DeepSeek
```

## 👥 P0 组织工作：种子用户选拔

**种子用户决定了项目的生死！**

### ✅ 理想种子用户画像

| 维度 | 标准 | 权重 |
|------|------|------|
| **意愿** | 主动报名，愿意尝鲜 | 40% |
| **耐心** | 能接受不完美，愿意提反馈 | 30% |
| **影响力** | 在团队中有一定影响力，能帮助推广 | 20% |
| **技术基础** | 会用基本工具，能独立解决简单问题 | 10% |

### ❌ 避免的种子用户

- 抱怨型：什么都不满意，只提问题不给建议
- 观望型："等你们做好了我再用"
- 完美主义型：要求一次就达到 99.9% 准确率
- 忙到没时间："我没空用，也没空反馈"

### 📊 种子用户规模

**建议：3-5 人，不宜太多！**

## 🕳️ P0 阶段三大陷阱

| 陷阱 | 表现 | 避坑指南 |
|------|------|---------|
| **一步到位症** | 上来就想做通用智能，什么功能都想加 | **渐进式**：P0 只做最核心的 3-5 个功能，稳定了再加 |
| **大模型万能论** | 什么都让大模型做，脚本能做的也不用 | **脚本优先**：脚本能做的绝对不用大模型，P0 目标 < 10% |
| **盲目扩大用户** | 刚跑起来就推给全公司用 | **种子用户**：先找 3-5 个种子用户用稳定了再说 |

## 📈 升级到 P1 的判断标准

满足以下全部条件，再升级到 P1：

1. ✅ 连续 7 天无故障运行
2. ✅ 脚本覆盖 ≥ 10 个常用任务
3. ✅ 缓存命中率 ≥ 50%
4. ✅ 种子用户满意度 ≥ 3.5/5
5. ✅ 用户主动要求更多自动化功能
