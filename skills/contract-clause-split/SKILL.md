# 合同条款拆分技能

> 智能拆分合同文档，按条款类型分类、提取、结构化输出。
> ⚠️ **安全规则**: 必须由用户主动触发，禁止任何自动化/定时调用！

## 🚀 触发条件

当用户需要：
- 拆分合同文档为结构化条款
- 提取特定类型的合同条款
- 对比不同版本合同的条款差异
- 生成合同条款摘要
- 审核合同条款风险

## ⚠️ 大模型使用安全规则

| 规则 | 要求 |
|------|------|
| **触发方式** | ✅ **必须用户主动发起**，禁止自动触发 |
| **推荐 Provider** | 🟢 **DeepSeek**（文档处理场景） |
| **文件大小限制** | 单次处理 ≤ 10MB，超过需拆分 |
| **调用间隔** | 每次条款分析间隔 3-8 秒（模拟人类阅读） |
| **并行处理** | 同时最多分析 1 份合同 |
| **工作时间** | 建议 9:00-18:00 执行 |

---

## 📋 功能清单

| 功能 | 说明 |
|------|------|
| 📄 **PDF/Word 解析** | 支持 PDF 和 Word 格式合同解析 |
| ✂️ **条款拆分** | 智能识别并拆分独立条款 |
| 🏷️ **条款分类** | 自动归类（基本信息、主体信息、标的条款、价格条款、履行条款、权利义务、违约责任、争议解决等） |
| 🔍 **关键信息提取增强** | 多模式冗余匹配，提取甲方/乙方、合同金额、税率、付款方式、交付地点、有效期、签署日期、统一社会信用代码等关键信息 |
| ⚠️ **风险识别** | 识别潜在风险条款并标注 |
| 📊 **Excel 多 Sheet 导出** | 6 Sheet 结构化输出：全部条款、分类视图、签署页、风险条款汇总、关键信息、原始文本 |
| 🖼️ **签署页图片提取** | 自动检测 PDF 最后3页，命中签署关键词则渲染为高清图片（300 DPI）并插入 Excel |

## 🔧 使用方法

### 1. 拆分单个合同

```bash
python3 scripts/split_contract.py --input 合同文件.pdf
```

### 2. 签署页提取选项

```bash
# 默认自动提取签署页（仅 PDF）
python3 scripts/split_contract.py --input 合同.pdf

# 跳过签署页提取
python3 scripts/split_contract.py --input 合同.pdf --no-signature
```

### 3. 终端预览结果

```bash
python3 scripts/split_contract.py --input 合同.pdf --preview
```

### 3. 仅提取特定类型条款

```bash
python3 scripts/split_contract.py --file 合同.pdf --types "违约责任,争议解决,保密条款"
```

### 4. 风险审核模式

```bash
python3 scripts/split_contract.py --file 合同.pdf --risk-check --output report.md
```

## 📑 条款分类体系

| 类别 | 说明 |
|------|------|
| 📌 **基本信息** | 合同名称、编号、签订日期、签订地点 |
| 👥 **主体信息** | 甲方/乙方名称、地址、联系方式、法定代表人 |
| 📋 **定义条款** | 合同中专用术语的定义 |
| 📝 **标的条款** | 合同标的、数量、质量标准 |
| 💰 **价格条款** | 合同金额、支付方式、支付时间 |
| ⏰ **履行条款** | 履行期限、地点、方式 |
| ⚖️ **权利义务** | 双方权利和义务条款 |
| 🛡️ **违约责任** | 违约情形、违约金、赔偿方式 |
| 🔒 **保密条款** | 保密义务、保密期限、保密范围 |
| 🤝 **争议解决** | 争议解决方式、管辖法院/仲裁委 |
| 🔄 **变更解除** | 合同变更和解除条件 |
| ✍️ **其他条款** | 不可抗力、通知方式、生效条件等 |
| 📎 **附件** | 合同附件清单和内容 |

## ⚙️ 配置说明

### 配置文件位置

`config/contract-split-config.json`

| 配置项 | 说明 |
|--------|------|
| `output.formats` | 支持的输出格式列表 |
| `clause_categories` | 条款分类体系定义 |
| `risk_keywords` | 风险关键词库 |
| `extraction_rules` | 关键信息提取规则 |
| `processing.batch_size` | 批处理大小（默认：1 条/次） |
| `processing.delay_between_clauses` | 条款处理间隔（秒） |

## 📝 标准处理流程 (SOP)

### 合同拆分流程

```
1. 用户上传合同文件并发起请求
   │
   ▼
2. 解析文档内容（PDF/Word）
   │  └── 每页解析间隔 1-2 秒
   │
   ▼
3. 预处理：清理格式、去除页眉页脚
   │
   ▼
4. 智能分句和条款边界识别
   │  └── 每条识别间隔 0.5-1 秒
   │
   ▼
5. 条款分类和标注
   │  └── 每条分类间隔 3-8 秒（模拟人类阅读判断）
   │
   ▼
6. 关键信息提取（金额、日期、主体等）
   │
   ▼
7. 风险条款识别（可选）
   │
   ▼
8. 生成结构化输出并展示给用户
```

## ⚠️ 注意事项

1. **用户确认原则**: 拆分结果仅供参考，重要合同建议人工复核
2. **敏感数据保护**: 合同内容不上传到第三方服务，全部本地处理
3. **格式限制**: 扫描件 PDF 需要先通过 OCR 处理（配合 pdf-ocr-extraction 技能）
4. **复杂合同**: 超过 50 页的合同建议分段处理
5. **版本兼容**: 支持 Word 2007+ (.docx) 和 PDF 1.4+ 格式
