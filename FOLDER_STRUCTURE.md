## 🎯 目录设计思想

### 1. 【分离】纲领 vs 执行
- **roadmap/**：只管「是什么、为什么」，规定方向和原则
- **skills/**：只管「怎么做」，提供工具和执行方法

### 2. 【统一】全局规范集中
- 所有智能体都必须遵循的规范，集中在 roadmap/ 和 team-rules/
- 避免每个 Agent 有自己的一套标准，造成混乱

### 3. 【可追溯】完整版本演进
- roadmap/ 保留所有历史版本，清晰记录从 v0.3 到 v2.0 的完整演进
- 便于复盘、对比、持续优化
