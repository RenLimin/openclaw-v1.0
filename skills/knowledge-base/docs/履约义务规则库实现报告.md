### 2.1 义务分类体系（8大类）

| 代码 | 义务类型 | 子类 | 风险等级 |

| OBJ-03 | 付款义务 | 预付款、进度款、验收款、质保金 | critical |
| OBJ-04 | 维保义务 | 保修期、保修范围、维保服务内容 | medium |
| OBJ-05 | 技术支持义务 | 支持方式、响应时间、支持范围 | low |
| OBJ-06 | 培训义务 | 培训内容、培训对象、培训时长 | low |
| OBJ-07 | 知识产权义务 | 权利归属、许可范围 | high |
| OBJ-08 | 违约责任 | 违约金、违约救济、违约解除 | critical |

## 四、技术特点

### 4.1 分类算法
- **加权关键词匹配**: 高权重(5分) + 中权重(2分)
- **特征句式匹配**: 模式匹配，每条10分
- **置信度加成**: 多关键词匹配、句式匹配加成
- **多分类支持**: 同一条款可触发多个义务类型

### 4.2 风险评估模型
- 基础风险: 根据义务类型预设
- 付款风险: 预付款>30%、无质保金
- 验收风险: 验收标准不明确
- 知识产权风险: 归属约定不明确
- 违约风险: 无违约金上限约定

### 4.3 标准比对规则
- 付款节奏比对
- 交付周期比对
- 保修期限比对
- 支持等级比对
- 知识产权条款比对

## 五、使用说明

### 5.1 基础使用

```python
from skills.knowledge_base.scripts.obligation_query import ObligationQuery

# 初始化
query = ObligationQuery()

# 处理单条款
obligations = query.process_clause("乙方应在30日内交付全部设备")

# 批量处理合同
result = query.process_contract([
    "条款1...",
    "条款2...",
    "条款3..."
])

# 导出
query.export_to_json(result, "obligations.json")
query.export_to_excel(result, "obligations.xlsx")
```

### 5.2 命令行测试

```bash
python3 skills/knowledge-base/scripts/obligation_query.py
```
