> **核心价值：专业化、可扩展、团队协作**

- 领域专家系统（法律、医疗、金融、代码等）

## 🏗️ KSA 三层能力架构（通用）

**Knowledge-Skill-Agent - 任何领域都适用的分层模型**

```
┌─────────────────────────────────────────────────────────┐
│  A - Agent 层 - 智能体外壳                               │
│  ✅ 任务理解、协调调度、上下文管理、结果汇总             │
│  📊 大模型调用占比：< 20%                                │
├─────────────────────────────────────────────────────────┤
│  S - Skill 层 - 技能执行层                               │
│  ✅ 原子能力、脚本库、工具链集成、专业处理逻辑           │
│  📊 大模型调用占比：< 10%                                │
├─────────────────────────────────────────────────────────┤
│  K - Knowledge 层 - 知识核心层                           │
│  ✅ 结构化知识库、规则引擎、向量检索、案例库             │
│  📊 大模型调用占比：< 5%（仅重排序）                     │
└─────────────────────────────────────────────────────────┘
```

### 通用原则

> **越往下，越不用大模型，越稳定，越便宜，越可控！**

## 👥 P2 人机协作模式：协作 + 监督

### 模式一：协作模式（人机各做擅长的部分）

```
用户
  ↓ 确认任务
智能体做 80% 标准化工作
  ↓ 提交中间结果
人做 20% 专业判断和修改
  ↓ 确认
智能体完成最后 10% 格式化工作
  ↓ 交付最终结果
```

**典型场景：合同审核**
- 智能体：格式检查、标准条款比对、风险点标注
- 人：模糊条款判断、商务决策、签字确认

### 模式二：监督模式（智能体全做，人做审核）

```
智能体 100% 自动处理
  ↓ 标注置信度
高置信度（>95%）→ 自动通过
中置信度（70%-95%）→ 人工抽样审核
低置信度（<70%）→ 100% 人工审核
```

**典型场景：代码评审、数据报表生成**

## 🔐 P2 完整风险管理体系

### 七大风险矩阵

| 风险类型 | P2 风险等级 | 典型表现 | 应对策略 |
|---------|------------|---------|---------|
| **幻觉风险** | 🟠 中高 | 编造不存在的条款、数据 | 多模型交叉验证 + 人工审核关键任务 |
| **数据泄露** | 🟠 中高 | 敏感数据被带出、权限滥用 | 沙箱隔离 + 数据脱敏 + 细粒度权限控制 |
| **成本失控** | 🔴 高 | 专业任务调用量大，费用激增 | MCP 预算控制 + 按智能体分账 + 用量告警 |
| **质量下降** | 🟡 中 | 知识库更新不及时、技能退化 | 基准测试 + 回归测试 + 版本管理 |
| **知识过时** | 🟡 中 | 政策法规更新，知识库没跟上 | 知识库定期更新机制 + 自动校验 |
| **工作替代** | 🟡 中 | 用户担心被替代，抵触使用 | 透明沟通 + 技能升级培训 + 岗位转型支持 |
| **权限滥用** | 🟠 中高 | 越权访问数据、执行危险操作 | 最小权限原则 + 操作审计 + 双人确认 |

### 🚨 P2 必须建立的安全机制

```json
{
  "safety": {
    "sandbox": {
      "mode": "per-agent",        // 每个智能体独立沙箱
      "network": "none",          // 默认禁用网络
      "filesystem": "read-only"    // 默认只读
    },
    "approval": {
      "highRiskActions": true,    // 高风险操作必须人工审批
      "multiPerson": true,         // 双人确认机制
      "auditLog": true             // 完整审计日志
    }
  }
}
```

## 🧠 轻量 RAG 架构（通用）

**SQLite + 全文检索 - 95% 场景够用，不用重型向量数据库**

```
用户提问
    ↓
SQLite 全文检索（10ms，不用大模型）
    ↓ 置信度 > 95%
直接返回结果 ✅（省大钱！）
    ↓ 置信度 70% ~ 95%
大模型重排序（仅排序，不生成内容）⚡
    ↓ 置信度 < 70%
降级到关键词匹配 + 提示用户补充信息 ❓
```

**P2 RAG 目标：检索层 95% 不用大模型！**

## 🕳️ P2 阶段三大陷阱

| 陷阱 | 表现 | 避坑指南 |
|------|------|---------|
| **知识垃圾场** | 什么都往知识库塞，又乱又杂 | **知识治理**：建立知识准入、更新、淘汰机制 |
| **智能体同质化** | 每个智能体能力都差不多，分工不明确 | **明确定位**：每个智能体必须有清晰的领域边界，能力不重叠 |
| **审核流于形式** | 什么都要人工审核，反而更累了 | **置信度分流**：高置信度自动过，中置信度抽样，低置信度全审 |
