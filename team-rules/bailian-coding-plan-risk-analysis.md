## 一、规则原文解读

### 1.1 核心禁令原文

> **严禁 API 调用**：仅限在编程工具（如 Claude Code、OpenClaw 等）中使用，禁止以 API 调用的形式用于**自动化脚本**、**自定义应用程序后端**或**任何非交互式批量调用**场景。将套餐 API Key 用于允许范围之外的调用将被视为违规或滥用，可能会导致订阅被暂停或 API Key 被封禁。

### 1.2 关键词拆解

| 关键词 | 定义 | 风险等级 |
|--------|------|---------|
| **仅限编程工具** | Claude Code、OpenClaw 等交互式编程助手 | ✅ 允许 |
| **自动化脚本** | 无需人工交互的脚本自动调用 | 🔴 高风险 |
| **应用后端** | 作为产品/服务的后端 API | 🔴 高风险 |
| **非交互式批量调用** | 批量、自动化、无人值守的调用 | 🔴 高风险 |

## 二、我们当前哪些行为涉嫌违规？

### 🔴 【高危】已明确违规的行为

#### 1. Cron 定时任务触发的邮件巡检
```
cron:38439607-589d-41b5-b775-467a4c33e8de Iris 邮件巡检
├── 触发方式：定时自动触发，无用户交互
├── 调用模式：非交互式批量
└── 结论：❌ 100% 违规 - 属于"非交互式批量调用"
```

#### 2. Cron 定时任务触发的 P0 邮件检查
```
cron:26ab8861-4fa7-45b8-bc74-d964e63ead56 Iris 邮件提醒 P0
├── 触发方式：定时自动触发，无用户交互
├── 调用模式：非交互式批量
└── 结论：❌ 100% 违规
```

#### 3. 子 Agent 自动调用链
```
Jerry 分派任务 → Ella/Oliver/Aaron/Iris 自动执行
├── 触发方式：Agent 间自动调用，无用户确认
├── 调用模式：半自动化
└── 结论：⚠️ 边界模糊，有风险 - 取决于是否"批量/无人值守"
```

### 🟡 【中风险】边界模糊的行为

#### 4. 技能训练中的批量处理
```
批量生成训练数据、批量评估、批量测试
├── 触发方式：可能是用户发起的，但执行过程全自动
├── 调用模式：批量调用
└── 结论：⚠️ 风险中等 - 取决于"批量"的定义和量级
```

#### 5. 长任务中的连续模型调用
```
一个复杂任务中连续调用 10+ 次模型
├── 触发方式：用户发起，但执行过程自动
├── 调用模式：连续、高频
└── 结论：⚠️ 低到中风险 - 频率过高可能触发风控
```

### ✅ 【低风险】明确允许的行为

#### 6. 用户直接交互的对话
```
用户: "帮我写个 Python 脚本"
Jerry: "好的，这是脚本..."
├── 触发方式：用户主动发起，有明确交互
├── 调用模式：交互式
└── 结论：✅ 完全允许 - 属于编程工具正常使用
```

#### 7. 用户确认后的单步操作
```
用户: "帮我检查一下合同"
Ella: "这是检查结果..."
├── 触发方式：用户明确指令
├── 调用模式：单任务、交互式
└── 结论：✅ 允许
```

## 三、阿里云如何检测违规？

### 3.1 检测维度推测

| 检测维度 | 正常模式 | 违规模式 |
|----------|---------|---------|
| **调用时间分布** | 工作时间、不均匀、有间隔 | 24 小时连续、均匀间隔、深夜高频 |
| **调用频率** | 低 QPS、有停顿 | 高 QPS、持续稳定 |
| **输入内容相似度** | 多样化、人类语言 | 模板化、高度相似、批量请求 |
| **交互模式** | 有来有回、有用户确认 | 单向调用、无反馈、链式调用 |
| **Token 消耗模式** | 波动大、任务驱动 | 稳定持续、匀速消耗 |
| **IP/UA 特征** | 人类使用环境 | 服务器环境、固定 IP |

### 3.2 我们的危险信号（已触发）

1. ✅ **定时 Cron** - 每小时 1 次、精准间隔 → 典型自动化特征
2. ✅ **24 小时运行** - 凌晨也在调用 → 非人类行为
3. ✅ **内容模板化** - 邮件巡检的 prompt 几乎完全相同 → 批量特征
4. ✅ **无人值守** - 调用时无用户在线 → 非交互式

**这 4 点加起来，不冻结才怪！** 🚨

## 四、整改方案

### 4.1 【立即执行】Cron 任务紧急整改

| Cron 任务 | 当前状态 | 整改方案 |
|-----------|---------|---------|
| 邮件巡检（非工作时间） | 🔴 运行中 | ❌ 立即删除，或改为手动触发 |
| P0 邮件提醒（工作时间） | 🔴 运行中 | ❌ 立即删除，或改为手动触发 |

> **建议**：邮件巡检改为 **用户主动触发** 或 **每日仅 1 次汇总**，且必须在工作时间，由用户确认后执行。

### 4.2 【Provider 隔离】高危任务只能用安全 Provider

| 任务类型 | 允许使用的 Provider | 禁止使用的 Provider |
|---------|---------------------|---------------------|
| Cron/自动化任务 | DeepSeek, Volcengine | ❌ bailian（所有模型） |
| 批量处理任务 | DeepSeek, Volcengine | ❌ bailian |
| 交互式对话任务 | 所有 Provider | - |

### 4.3 【调用行为优化】模拟人类使用模式

| 优化项 | 当前 | 整改后 |
|--------|------|--------|
| 连续调用间隔 | ~0ms（立即） | ≥ 3s 随机延迟 |
| 单任务最大调用次数 | 无限制 | ≤ 5 次 |
| 每日最大调用次数（bailian） | 无限制 | ≤ 50 次 |
| 调用时间窗口 | 24 小时 | 仅 9:00-18:00 |
| 连续调用之间 | 无停顿 | 每 3 次调用后强制停顿 30s |

### 4.4 【熔断机制】bailian 二级熔断

```
bailian 调用
   │
   ├── 检测：是否在白名单时间（9:00-18:00）？
   │   └── 否 → 拒绝，切换到 DeepSeek
   │
   ├── 检测：今日调用次数 > 50？
   │   └── 是 → 熔断，切换到 DeepSeek
   │
   ├── 检测：是否为自动触发任务（Cron）？
   │   └── 是 → 拒绝，切换到 DeepSeek
   │
   └── 检测：连续调用 > 3 次？
       └── 是 → 强制停顿 30s
```

## 五、bailian 使用白名单 vs 黑名单

### ✅ 白名单（允许使用 bailian）

1. 用户主动发起的对话
2. 代码补全、代码解释、Debug 辅助
3. 单步问题解答
4. 用户明确确认后的单次任务

### ❌ 黑名单（绝对禁止使用 bailian）

1. ❌ Cron 定时触发的任何任务
2. ❌ 批量数据处理
3. ❌ 自动化脚本调用
4. ❌ 子 Agent 链式自动调用
5. ❌ 22:00 - 次日 8:00 的任何调用
6. ❌ 单日单账号调用超过 50 次
7. ❌ 连续调用超过 3 次无停顿
