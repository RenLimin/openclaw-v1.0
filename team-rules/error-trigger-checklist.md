# 🔴 敏感操作前置检查清单 v1.0
> **强制执行！** 做任何以下操作前，必须逐项检查确认！
> **触发时机：** 执行操作前，必须打开本文件逐项核对！

---

## 📋 第一类：文档变更类操作

### 🔍 检查点 1：版本号同步
**在修改任何带版本号的文档后，提交前必须确认：**
- [ ] ✅ 文件头部的版本号已更新（严格遵循 SemVer）
- [ ] ✅ 文件名中的版本号已同步更新（如果有）
- [ ] ✅ 文档中所有引用此版本号的地方已同步更新

### 🔍 检查点 2：变更摘要
- [ ] ✅ 在文档更新日志中清晰记录本次变更内容
- [ ] ✅ 变更类型与版本号匹配（PATCH/MINOR/MAJOR）

---

## 📋 第二类：GitHub 提交类操作

### 🔍 检查点 1：用户批准确认
**在执行任何 `git push` 前，必须确认：**
- [ ] ✅ 已获得用户明确的「提交」指令批准
- [ ] ✅ 没有未获批准的推送操作
- [ ] ✅ 推送内容与用户批准内容一致

### 🔍 检查点 2：提交内容检查
- [ ] ✅ 只包含计划内的变更，没有意外文件
- [ ] ✅ 提交信息格式规范，版本号与变更内容匹配
- [ ] ✅ 没有泄露敏感信息或不需要的文件

---

## 📋 第三类：智能体团队创建类操作

### 🔍 检查点 1：前置分析确认
- [ ] ✅ 已完成业务目标深度分析（使用 business-goal-analyzer）
- [ ] ✅ 输出了完整的 7 份标准交付物
- [ ] ✅ 用户已明确确认分析结果
- [ ] ✅ 已阅读 roadmap/ 下所有纲领性指南

---

## 📋 第四类：配置修改类操作

### 🔍 检查点 1：核心配置修改检查
- [ ] ✅ 已确认修改的影响范围
- [ ] ✅ 用户批准修改内容
- [ ] ✅ 已记录版本变更说明

---

## 🔄 使用流程（强制执行！）

触发敏感操作
    ↓
