## 🏗️ 核心架构原则（强制执行）

### ✅ 原则1：脚本程序为主，大模型仅做增强
| 任务类型 | 实现方式 | 说明 |
|---------|---------|------|
| 数据查询/统计/报表 | 100% Python脚本 + SQL | 纯确定性逻辑，绝不调用大模型 |
| 规则匹配/格式校验 | 100% Python脚本 | 规则引擎实现 |
| 合同条款定位/字段提取 | OCR引擎 + 正则表达式为主 | 大模型仅做兜底校验和置信度标注 |
| 风险识别 | 规则引擎识别80%常见风险 | 大模型仅识别剩余20%复杂/模糊场景 |
| 洞察生成/自然语言问答 | 大模型 | 需要理解和推理的场景 |

> **目标**：大模型调用量占比 ≤20%，80%业务逻辑靠确定性代码完成

### ✅ 原则2：统一内置轻量数据库（SQLite）
**统一数据库文件**：`data/delivery_management.db`

```
核心数据表设计：
├── contract_contracts        # 合同主表
├── contract_items           # 合同分项表
├── contract_risk            # 合同风险记录表
├── projects                  # 项目主表
├── project_tasks            # 任务表
├── project_costs           # 成本明细表
├── work_hours              # 工时表
├── requirements            # 需求表
├── test_cases             # 测试用例表
├── knowledge_docs          # 知识库文档表
├── meeting_minutes         # 会议记录表
├── action_items           # 待办任务表
├── prompt_templates        # Prompt模板库
└── llm_call_logs          # 大模型调用日志（成本统计用）
```

### ✅ 原则3：标准化接口设计
每个技能模块必须提供3种调用方式：
```bash
# 1. CLI命令行（脚本化调用，默认不调用大模型）
python -m skills.contract_audit --file contract.pdf

# 2. 开启大模型增强（显式指定）
python -m skills.contract_audit --file contract.pdf --enable-llm
```

```python
# Python API 调用
from skills.contract_audit import audit_contract

# 纯脚本模式（默认）
result = audit_contract("contract.pdf")

# 大模型增强模式（显式开启）
result = audit_contract("contract.pdf", enable_llm=True)
```
