### 2.1 优先级总览

```
                            ╱╲
                           ╱══╲
                          ╱════╲
                         ╱══════╲
                        ╱════════╲
                       ╱══════════╲
                      ╱════════════╲
                     ╱══  最高优先级  ══╲
                    ╱══════════════════╲
                   ╱════════════════════╲
                  ╱══════════════════════╲
                 ╱════════════════════════╲
                ╱══════════════════════════╲
               ╱════════════════════════════╲
              ╱══════════════════════════════╲
             ╱════════════════════════════════╲

    🔝 6. SKILL.md (工作区私有技能)
            ~/.openclaw/workspace/skills/<name>/SKILL.md
            ← 同名技能覆盖所有下层 ←

    🔝 5. 工作区 Bootstrap 文件
            AGENTS.md, SOUL.md, USER.md, TOOLS.md, IDENTITY.md
            ← 注入系统提示词，模型层生效 ←

    🔝 4. 单个智能体配置 (agents.list[].*)
            ← 覆盖智能体默认值 ←

    🔝 3. 智能体默认配置 (agents.defaults.*)
            ← 全局智能体基线 ←

    🔝 2. 主配置文件 (openclaw.json)
            ← 所有全局设置入口 ←

    🔝 1. 环境变量 (OPENCLAW_*)
            ← 覆盖 openclaw.json 中的字段 ←

    🔝 0. 内置默认值 (编译在二进制中)

             ════════════════════════════
                 最低优先级
```

### 2.2 优先级冲突解决规则

```
┌─────────────────────────────────────────────────────────────────┐
│                    优先级冲突解决规则表                          │
├───────────────────────────────────────────────┬─────────────────┤
│ 冲突场景                                       │  获胜方         │
├───────────────────────────────────────────────┼─────────────────┤
│ 环境变量 vs openclaw.json                      │  环境变量 ✅    │
│ openclaw.json vs AGENTS.md                     │  AGENTS.md ✅   │
│ agents.list[x] vs agents.defaults              │  list[x] ✅     │

│ AGENTS.md vs SOUL.md (同层级)                   │  都生效，顺序  │

│ tools.deny vs tools.allow                       │  deny ✅       │
└───────────────────────────────────────────────┴─────────────────┘
```

## 三、SKILL.md 多层级加载机制

### 3.1 SKILL.md 6 层加载路径与优先级

```
╔══════════════════════════════════════════════════════════════════╗
║                    SKILL.md 六层级加载机制                       ║
║                                                                 ║
║   优先级从高 → 低，同名技能上层覆盖下层                          ║
╚══════════════════════════════════════════════════════════════════╝

  🔝 优先级 6 (最高)
  ┌───────────────────────────────────────────────────────────────┐
  │  当前工作区技能 (Per-Workspace Private)                       │
  │  ~/.openclaw/workspace/skills/<skill-name>/SKILL.md          │
  │  + 每个智能体独立工作区也有自己的 skills 目录                   │
  │  = 某个项目特有的自定义技能，完全覆盖所有下层                  │
  └───────────────────────────────────────────────────────────────┘
                          ↓ 向下覆盖

  🔝 优先级 5
  ┌───────────────────────────────────────────────────────────────┐
  │  项目级 Agent 共享技能                                        │
  │  ~/.openclaw/workspace/.agents/skills/<skill-name>/SKILL.md  │
  │  = 多智能体协作时，整个项目共享的技能                          │
  └───────────────────────────────────────────────────────────────┘
                          ↓ 向下覆盖

  🔝 优先级 4
  ┌───────────────────────────────────────────────────────────────┐
  │  用户级共享技能                                               │
  │  ~/.agents/skills/<skill-name>/SKILL.md                      │
  │  = 同一主机用户下所有 OpenClaw 实例都能使用的技能             │
  └───────────────────────────────────────────────────────────────┘
                          ↓ 向下覆盖

  🔝 优先级 3
  ┌───────────────────────────────────────────────────────────────┐
  │  系统级共享技能                                               │
  │  ~/.openclaw/skills/<skill-name>/SKILL.md                    │
  │  = 整个 OpenClaw 系统全局共享的技能                           │
  └───────────────────────────────────────────────────────────────┘
                          ↓ 向下覆盖

  🔝 优先级 2
  ┌───────────────────────────────────────────────────────────────┐
  │  内置捆绑技能 (Bundled Skills)                                │
  │  OpenClaw 二进制自带的技能 (weather, github 等)               │
  │  = 开箱即用的内置技能，优先级仅高于外部目录                   │
  └───────────────────────────────────────────────────────────────┘
                          ↓ 向下覆盖

  🔝 优先级 1 (最低)
  ┌───────────────────────────────────────────────────────────────┐
  │  额外技能目录 (skills.load.extraDirs 配置)                   │
  │  e.g. /opt/company-skills/, /shared/team-skills/             │
  │  = 企业/团队统一管理的技能库，所有实例都可使用                │
  └───────────────────────────────────────────────────────────────┘


  ═══════════════════════ 技能白名单关卡 ════════════════════════
                          ↓ ↓ ↓
  ┌───────────────────────────────────────────────────────────────┐
  │  白名单过滤 (两层)                                             │
  │  1. agents.defaults.skills → 所有智能体默认可见的技能          │
  │  2. agents.list[x].skills → 单个智能体可见的技能              │
  │                                                               │
  │  ❌ 不在白名单 = 即使文件存在也不会被加载！                   │
  └───────────────────────────────────────────────────────────────┘
                          ↓
  ┌───────────────────────────────────────────────────────────────┐
  │  依赖检查 (三层)                                              │
  │  bins[]    → 检查二进制是否存在 (pdftotext, tesseract 等)   │
  │  env[]     → 检查环境变量是否设置                            │
  │  config[]  → 检查配置项是否存在/启用                         │
  │                                                               │
  │  ❌ 任何一项不满足 = 技能不会被加载                          │
  └───────────────────────────────────────────────────────────────┘
                          ↓
  ┌───────────────────────────────────────────────────────────────┐
  │  ✅ 最终生效的技能集                                          │
  │  = 6 层路径加载 - 同名覆盖 - 白名单过滤 - 依赖检查           │
  └───────────────────────────────────────────────────────────────┘
```

## 四、多智能体配置继承关系

### 4.1 单个智能体配置继承链

```
╔══════════════════════════════════════════════════════════════════╗
║              单个智能体配置继承链 (以 ella 为例)                 ║
╚══════════════════════════════════════════════════════════════════╝

          ┌─────────────────────────────────────────┐
          │     0. 内置默认值 (编译在二进制)       │
          │     (所有字段的 fallback 值)          │
          └──────────────┬──────────────────────────┘
                         │ inherit
                         ▼
          ┌─────────────────────────────────────────┐
          │  1. openclaw.json 全局默认             │
          │  agents.defaults.model.primary        │
          │  agents.defaults.sandbox.mode         │
          │  agents.defaults.heartbeat.every      │
          │  agents.defaults.skills = [A, B, C]   │
          └──────────────┬──────────────────────────┘
                         │ inherit + 部分字段可覆盖
                         ▼
          ┌─────────────────────────────────────────┐
          │  2. openclaw.json 单个智能体配置       │
          │  agents.list[x].model.primary         │
          │  agents.list[x].sandbox.mode          │
          │  agents.list[x].skills = [D, E]       │
          │                                      │
          │  ⚠️ 注意：数组/对象字段 = 完全覆盖  │
          │  skills: [A,B,C] → 丢失！完全覆盖   │
          └──────────────────────┴────────────────
```

### 4.2 工作区文件独立配置机制

```
┌──────────────────────────────────────────────────────────────────────┐
│           多智能体工作区独立配置机制                               │
│                                                                  │
│  main 主智能体工作区                               ella 工作区    │
│  ├── AGENTS.md ← Ella 继承 main 的吗？不！完全独立]              │
│  ├── SOUL.md    ← 每个智能体有自己独立的人格设置                 │
│  ├── TOOLS.md   ← 每个智能体有自己独立的工具规范                │
│  └── skills/    ← 每个智能体可以有私有的技能                    │
│                                                                  │
│  关键点：每个智能体读取自己的 workspace，互不影响                   │
│  效果：每个智能体人格、技能、行为完全独立设置                      │
│                                                                  │
│  [独立性]                                 Ella 工作区配置       │
│  ┌───────────────────────────────────────────────────────────────│
│  Ella 的 AGENTS.md 中"禁止调用 exec"] 与 main 智能体无关         │
│  Oliver 的 SOUL.md 可以设置不同的人格风格                          │
│  每个智能体独立加载自己的 bootstrap 文件                          │
│  每个智能体独立加载自己的私有技能                                  │
│                                                                   │
│  最终效果：每个智能体行为完全独立隔离，互不影响                     │
│  适合场景：不同智能体处理不同安全等级的任务                          │
│                                                                   │
│  合同智能体 Ella 使用[严格审批] SOP，项目智能体 Oliver 使用[快速响应]
│ 巡检智能体 Iris 使用只读配置，不同智能体完全独立配置
```

## 五、运行时数据流全景图

### 5.1 一次智能体调用的完整配置链

```
 用户发消息 → OpenClaw Gateway
                  │
                  ▼
  ┌──────────────────────────────────────────────────────────────────────┐
  │  步骤 1: 通道消息接收                                               │
  │  - 读取 channels.wecom/whatsapp/telegram 配置                      │
  │  - 检查白名单 allowFrom                                           │
  └──────────────────────────────────────────────────────────────────────┘
                  │
                  ▼
  ┌──────────────────────────────────────────────────────────────────────┐
  │  步骤 2: bindings 路由匹配                                          │
  │  - 根据 channel + peer.id 匹配绑定规则                             │
  │  - 决定消息交给哪个智能体处理                                      │
  └──────────────────────────────────────────────────────────────────────┘
                  │
                  ▼
  ┌──────────────────────────────────────────────────────────────────────┐
  │  步骤 3: 会话上下文加载                                             │
  │  - 读取 session.dmScope (per-channel-peer)                       │
  │  - 加载历史会话记录 (sessions.jsonl)                                │
  │  - 检查会话重置策略 (daily, idle)                                │
  └──────────────────────────────────────────────────────────────────────┘
                  │
                  ▼
  ┌──────────────────────────────────────────────────────────────────────┐
  │  步骤 4: 智能体配置加载                                             │
  │  - 读取 agents.defaults.*                                         │
  │  - 应用 agents.list[id] 覆盖 (合并)                                │
  │  - 确定模型、沙箱、工具权限、技能白名单                            │
  └──────────────────────────────────────────────────────────────────────┘
                  │
                  ▼
  ┌──────────────────────────────────────────────────────────────────────┐
  │  步骤 5: 工作区 Bootstrap 文件注入                                 │
  │  - 检查 contextInjection 策略 (always/continuation-skip/never)     │
  │  - 读取 AGENTS.md, SOUL.md, USER.md, TOOLS.md, IDENTITY.md        │
  │  - 检查 MEMORY.md, HEARTBEAT.md (如果是心跳触发)                  │
  │  - 应用字符限制 (bootstrapMaxChars, bootstrapTotalMaxChars)       │
  │  - 生成系统提示词的基础部分                                        │
  └──────────────────────────────────────────────────────────────────────┘
                  │
                  ▼
  ┌──────────────────────────────────────────────────────────────────────┐
  │  步骤 6: 技能加载与注入                                             │
  │  - 扫描 6 层技能目录 (按优先级从高到低)                             │
  │  - 白名单过滤 (检查 agents.defaults.skills 及 agents.list[id].skills)│
  │  - 依赖检查 (bins, env, config)                                      │
  │  - 注入技能说明到系统提示词 (maxSkillsPromptChars限制)            │
  └──────────────────────────────────────────────────────────────────────┘
                  │
                  ▼
  ┌──────────────────────────────────────────────────────────────────────┐
  │  步骤 7: 工具权限最终计算                                           │
  │  - 确定工具 profile (full/coding/messaging/minimal)                │
  │  - 应用 tools.allow/deny 全局白/黑名单                             │
  │  - 应用 agents.defaults.tools.allow/deny                            │
  │  - 应用 agents.list[id].tools.allow/deny                           │
  │  - 最终检查沙箱权限是否允许 exec 等危险工具                         │
  └──────────────────────────────────────────────────────────────────────┘
                  │
                  ▼
  ┌──────────────────────────────────────────────────────────────────────┐
  │  步骤 8: 模型调用准备                                             │
  │  - 加载模型配置 models.providers.<provider>                      │
  │  - 检查 fallback 链 (primary → 降级模型)                          │
  │  - 从 agentDir 读取 auth-profiles.json 获取 API Key             │
  │  - 应用模型参数 (temperature, maxTokens 等)                        │
  └──────────────────────────────────────────────────────────────────────┘
                  │
                  ▼
  ┌──────────────────────────────────────────────────────────────────────┐
  │  步骤 9: 执行与工具调用                                             │
  │  - 沙箱创建检查 (sandbox.mode/sandbox.scope]                    │
  │  - 工具参数验证                                                    │
  │  - 工具执行 & 结果返回                                             │
  │  - 子代理 spawn (检查 maxSpawnDepth)                               │
  └──────────────────────────────────────────────────────────────────────┘
                  │
                  ▼
  ┌──────────────────────────────────────────────────────────────────────┐
  │  步骤 10: 结果返回与记忆存储                                       │
  │  - 消息格式化返回用户                                            │
  │  - 会话历史保存 (sessions/<id>.jsonl)                            │
  │  - 记忆提取与存储 (memory/YYYY-MM-DD.md)                        │
  │  - 梦境索引更新 (如 dreaming 开启)                               │
  └──────────────────────────────────────────────────────────────────────┘
```

## 六、典型场景追踪

### 6.1 场景 A: "为什么我修改了 AGENTS.md，智能体行为没变？"

```
问题诊断路径图

用户修改了 AGENTS.md
    │
    ▼
┌─────────────────────────────────────────────────┐
│ 检查 1: contextInjection 策略                   │
│ agents.defaults.contextInjection = ?           │
└─────────────────────────────────────────────────┘
    │ 值是什么？
    ├─ 值 = never → 完全不注入，修改当然无效 ❌
    ├─ 值 = continuation-skip → 看是不是"连续对话"
    │   (上次回复直接继续的对话跳过注入)
    │   执行 /reset 重置会话后生效 ✅
    └─ 值 = always → 应该每次都注入，继续检查
        │
        ▼
┌─────────────────────────────────────────────────┐
│ 检查 2: 文件大小与字符限制                      │
│ 文件 AGENTS.md 大小？                          │
└─────────────────────────────────────────────────┘
    │ 是否超过了 bootstrapMaxChars？(默认 12KB)
    ├─ 是 → 文件被截断，后面的内容没生效 ⚠️
    │    拆分到多个文件或精简内容
    └─ 否 → 大小正常，继续检查
        │
        ▼
┌─────────────────────────────────────────────────┐
│ 检查 3: 智能体是否用了独立工作区？              │
│ 修改的是 main 的 workspace，还是 ella 的？     │
└─────────────────────────────────────────────────┘
    │ 修改错了工作区路径
    ├─ 是 → 用户以为修改了 ella，实际 ella 有独立工作区
    │    找到正确的路径修改
    └─ 否 → 路径正确，继续检查
        │
        ▼
┌─────────────────────────────────────────────────┐
│ 检查 4: 是否有更高优先级的覆盖？                │
│ 例如：SKILL.md 中有更高优先级的规则覆盖        │
└─────────────────────────────────────────────────┘
    ├─ 技能中的规则优先级比 AGENTS.md 更高
    └─ 子代理的规则可能覆盖主智能体规则

✅ 解决：执行 /reset 重置会话 → 新上下文立即生效
```

```
└─────────────────────────────────────────────────┘
    │
    └─ 重载后查看日志确认技能已加载

```

### 6.3 场景 C: "子代理 spawn 失败"

```
问题诊断路径图

    │

│ 子代理目标智能体在 subagents.allowAgents 吗？  │
└─────────────────────────────────────────────────┘
    │
    ├─ 不在白名单 → 添加到允许列表
    └─ 在白名单 → 继续检查
        │
        ▼
┌─────────────────────────────────────────────────┐
│ 检查 3: 父智能体有 sessions_spawn 工具权限吗？ │
│ tools.allow 中包含 sessions_spawn 吗？         │
└─────────────────────────────────────────────────┘
    │
    ├─ 工具权限不足 → 确保 coding profile 或显式 allow
    └─ 有权限 → 继续检查
        │
        ▼

│ 是否已经达到了子代理并发上限？                  │
└─────────────────────────────────────────────────┘
    │
    ├─ 并发已满 → 等待其他子代理结束或提高限制
    └─ 未达上限 → 检查模型配置是否正确
        │
        ▼
┌─────────────────────────────────────────────────┐
│ 检查 5: 目标智能体模型配置                      │
│ 目标智能体 model 配置正确吗？API Key 有效吗？ │
```

## 七、配置变更影响扩散图

### 7.1 修改某个配置字段的影响范围

```
┌───────────────────────────────────────────────────────────────────────┐
│                    配置字段修改影响扩散图                               │
└───────────────────────────────────────────────────────────────────────┘

修改的字段位置                  影响范围
─────────────────────────────────────────────────────────────────────
1. gateway.port            → [全部] 网关重启，所有连接断开
   gateway.bind
   gateway.auth

2. agents.defaults.model.primary → [所有智能体] 默认模型改变
   agents.defaults.sandbox.mode → [所有智能体] 沙箱策略改变
   agents.defaults.heartbeat    → [所有智能体] 心跳行为改变
   agents.defaults.skills       → [所有智能体] 可见技能集改变

3. agents.list[x].model        → [单个智能体] 仅该智能体模型改变
   agents.list[x].skills       → [单个智能体] 仅该智能体技能改变
   agents.list[x].sandbox      → [单个智能体] 仅该智能体沙箱改变

4. skills.entries.<name>.env  → [使用该技能的所有智能体]
   skills.entries.<name>.enabled → 所有加载该技能的智能体

5. workspace/AGENTS.md         → [该工作区所属的智能体]
   workspace/SOUL.md
   workspace/TOOLS.md
   workspace/MEMORY.md

6. skills/<name>/SKILL.md     → [所有加载该技能的智能体]
                                  受白名单过滤，不是所有智能体都受影响

7. bindings[].agentId          → [匹配该规则的消息路由改变]
   bindings[].match            → 后续消息会路由到不同智能体

8. channels.<chan>.allowFrom   → [该通道所有消息]
                                  白名单内用户才能发消息

9. plugins.entries.<p>.enabled → [全部] 需要重启网关，插件卸载/加载
   plugins.entries.<p>.config

10. cron.maxConcurrentRuns     → [所有 Cron 任务] 调度策略改变
    cron.jobs.json 修改        → 特定 Cron 任务改变

11. hooks.token                → [所有 Webhook] 调用者需要更新 token
    hooks.mappings             → [特定 Webhook] 路由行为改变
```
