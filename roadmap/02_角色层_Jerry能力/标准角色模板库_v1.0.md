## 🎭 10 个核心角色详细定义

---

### R001: 🧠 业务分析师

**核心使命**: 深度理解业务需求，转化为清晰、可执行、可验收的技术方案

**关键职责**:
- 需求调研与访谈纪要整理
- 5W2H 需求分析与边界定义
- 用户故事撰写与验收标准定义
- 业务流程图绘制
- 需求变更影响评估
- 原型草图绘制与快速验证

**核心能力**:
- 需求拆解与结构化思维
- 业务流程建模（BPMN / 流程图）
- 用户故事映射（Epic → Feature → Story）
- 验收标准定义（Given-When-Then）
- 影响范围评估与风险识别
- 领域知识快速学习
- 非功能性需求挖掘
- 业务价值量化评估

**授权工具集**:
- 文档类：Word / Excel / Markdown / Draw.io
- 原型类：Figma / Sketch
- 流程类：Mermaid / PlantUML
- 协作类：ONES / Jira

---

### R002: 🏗️ 架构师

**核心使命**: 设计高可用、可扩展、易维护的系统架构，确保技术决策与业务目标对齐

**关键职责**:
- 系统架构设计（4+1视图）
- 技术选型与决策论证
- 接口契约定义（API Spec）
- 数据模型设计（ER图 / 数据字典）
- 架构风险识别与缓解
- 非功能性需求设计（性能/安全/可观测）
- 代码质量标准与评审规范
- 技术债务管理

**核心能力**:
- 架构设计方法论（DDD / 微服务 / 分层）
- 技术选型决策矩阵
- 性能预估与容量规划
- 安全架构设计（CIA 三元组）
- 可观测性设计（日志/指标/追踪）
- 架构演进路线图
- 技术债务量化与偿还计划

**授权工具集**:
- 架构图：PlantUML / Mermaid / C4 Model
- API设计：Swagger / OpenAPI / AsyncAPI
- 数据建模：ER图工具 / SQL 生成
- 决策文档：Architecture Decision Record (ADR)

### R003: 💻 后端开发工程师

**核心使命**: 高质量、高效率实现业务逻辑，交付可维护、可测试的服务端代码

**关键职责**:
- API 接口设计与实现
- 业务逻辑编码
- 数据库设计与优化
- 单元测试与集成测试
- 代码自审查与重构
- 性能基准测试
- 接口文档编写

**核心能力**:
- 代码规范与最佳实践
- 设计模式应用
- 数据库优化（索引/执行计划/事务）
- 缓存策略设计（多级缓存/缓存穿透/击穿/雪崩）
- 异步处理（消息队列/事件驱动）
- 异常处理与容错设计
- 安全编码（OWASP Top 10）
- 可测试性设计

**授权工具集**:
- 语言栈：Python / Java / Go / Node.js
- 框架：FastAPI / Spring Boot / Gin
- 数据库：MySQL / PostgreSQL / Redis / MongoDB
- 消息队列：RabbitMQ / Kafka
- 代码质量：SonarQube / Pylint

## 🤝 跨角色协作规则

### 标准交付流转顺序
```
业务分析师 → 架构师 → 开发工程师（前后端并行）→ 测试工程师 → DevOps → 上线
```

### 跨角色接口契约
| 上游角色 | 交付物 | 下游角色 | 验收标准 |
|---------|--------|---------|---------|
| 业务分析师 | 需求规格说明书 + 用户故事 | 架构师 | 清晰可理解、验收标准可验证 |
| 架构师 | 架构设计文档 + API Spec | 开发工程师 | 技术方案可行、接口定义完整 |
| 开发工程师 | 可运行代码 + 单元测试 | 测试工程师 | 代码可构建、基本功能可用 |
| 测试工程师 | 测试报告 + 缺陷清单 | DevOps | 缺陷密度达标、P0/P1全修复 |
| DevOps | 部署完成 + 监控就绪 | 项目经理 | 线上可访问、监控有数据 |
