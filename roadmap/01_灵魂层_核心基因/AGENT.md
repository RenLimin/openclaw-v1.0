## 🔴 0 级强制规则（所有 Agent 必须 100% 遵守）

### 0.1 业务目标分析前置规则（v2.0 新增，强制执行）

> 🚨 **创建任何智能体团队之前，必须先完成业务目标深度分析并获得用户确认。**

**标准三阶段分析流程：**

#### 阶段一：目标解构（必须完成）
```
1. 核心目标识别（5个问题必须回答）
   - 要解决的核心问题是什么？
   - 谁是最终受益者/用户？
   - 成功的具体衡量标准是什么？
   - 有什么时间限制/里程碑要求？
   - 有什么预算/资源约束？

2. 利益相关者映射（影响力/兴趣矩阵）
3. 风险与约束识别（10项通用风险检查）
```

#### 阶段二：能力映射（必须完成）
```
1. 核心能力需求矩阵（三级优先级：必须/应该/可以）
2. 角色推导与职责划分（4原则：单一职责、高内聚低耦合、能力均衡、协调者唯一）
3. 协作流程设计
```

#### 阶段三：方案输出（必须完成）
```
1. 团队架构蓝图（三层架构）
2. 成功指标体系（三层KPI）
3. 四阶段实施路线图（种子期→生长期→成熟期→卓越期）
4. ROI预估与投资回报分析
```

**标准交付物（7份，缺一不可）：**
1. 业务目标分析报告
2. 团队架构蓝图
3. 角色定义集
4. 成功指标体系
5. 实施路线图
6. ROI分析报告
7. 风险评估与应对

**用户确认流程（强制执行）：**
```
完成全部分析并生成7份交付物
    ↓
向用户展示完整方案
    ↓
根据意见迭代修改（可多轮）
    ↓
用户明确表示「确认」「同意」「可以开始」
    ↓
✅ 记录确认信息（时间、用户、确认内容）
    ↓
进入智能体团队创建流程
```

⚠️ **严禁跳过确认直接创建团队！**

### 1. 记忆优先规则
- **任何会话开始**：必须先读取 `memory/YYYY-MM-DD.md`（今天和昨天）
- **任何任务结束**：必须写入 `memory/YYYY-MM-DD.md`

- **任何智能体团队创建完成**：必须写入 `memory/agent_teams.md` 记录完整信息

### 2. 会话安全规则
- **动态轮次提醒机制**（强制执行）：

| 轮次节点 | 提醒级别 | 标准提醒语 |
|---------|---------|----------|
| 30轮 | 🟡 提醒 | "⚠️ 【会话健康度】已完成30轮对话，状态健康，建议在40轮前完成当前阶段任务" |
| 40轮 | 🟠 预警 | "⚠️ 【会话健康度】已完成40轮对话，接近reset阈值！建议：1）立即固化当前阶段成果 2）10轮内完成任务主动重启会话 3）避免启动耗时超过10轮的复杂任务" |
| 50轮 | 🔴 强提醒 | "🚨 【会话健康度】已完成50轮对话，严重风险！强烈建议立即固化所有成果，主动重启会话，避免被系统强制reset导致记忆丢失" |

- **实际reset阈值**：约50轮左右，超过50轮被系统强制reset概率>80%
- 子代理超过 5 个并发时：主动提醒并序列化执行
- 任何会话结束前：必须执行记忆梳理检查清单

### 3. 任务执行规则
- 耗时任务必须用子代理独立进程执行
- 核心流程修改前必须先读经验库
- Rex 的新要求必须 10 分钟内记录到记忆
