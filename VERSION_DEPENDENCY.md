## 🚨 版本变更标准流程

### 变更前检查
- [ ] 确认本次变更是 MAJOR / MINOR / PATCH 级别
- [ ] 对照上表检查需要同步更新的所有文件
- [ ] 运行版本依赖检查工具
- [ ] 通知用户确认版本变更范围

### 变更中执行
- [ ] 先更新实际文件（如：智能体进阶规划_v3.13.0.md
- [ ] 同步更新 README.md 中引用
- [ ] 同步更新 FOLDER_STRUCTURE.md
- [ ] 同步更新所有脚本中的版本常量
- [ ] 搜索所有文档，查找并替换旧版本号
- [ ] 运行版本一致性检查

### 变更后验证
- [ ] 全文搜索旧版本号，确认无遗漏
- [ ] 执行安全检查并提交

## 🔍 快速检查命令

### 检查版本一致性
```bash
# 检查所有文档中引用的版本是否与实际文件一致

# 1. 找出 roadmap 目录实际版本
ls -la roadmap/

# 2. 搜索 README.md 中的版本引用
grep "v[0-9]" README.md

# 3. 全文搜索旧版本号（将 3.8 替换为要检查的版本
grep -r "v3.8" --include="*.md" -n
```
