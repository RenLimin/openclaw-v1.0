## 🎯 版本体系架构


## 🔍 版本一致性检查工具

### 一键检查命令

```bash
cd roadmap/
find . -name "*.md" -path "*/_archive/*" -prune -o -name "*.md" -print | while read file; do
    # 提取文件名中的版本号
    version=$(echo "$file" | grep -oE "v[0-9]+\.[0-9]+\.[0-9]+" | head -1)
    if [ -z "$version" ]; then
        echo "❌ $file (无版本号)"
    elif [ "$version" != "v4.1.0" ]; then
        echo "⚠️  $file ($version != v4.1.0)"
    else
        echo "✅ $file ($version)"
    fi
done
```

## 💡 设计哲学

### 为什么采用「全局主版本 + 模块小版本」？

| 方案 | 优点 | 缺点 | 适用 |
|------|------|------|------|
| **完全独立版本** | 各模块自由演进 | 版本混乱，不知道该用哪个 | ❌ 不采用 |
| **完全统一版本** | 简单清晰，所有模块同号 | 模块没变化也要改版本号 | ❌ 不采用 |
| ✅ **全局主版本 + 模块小版本** | 全局清晰，模块灵活 | 文件名稍长 | ✅ **最佳方案** |

### 核心优势

1. **一眼可知兼容性**：看到 v4.1.0 前缀，就知道所有模块都属于同一体系，互相兼容
2. **模块可独立演进**：模块内部的小迭代不影响全局主版本
3. **历史追溯清晰**：通过版本号可以快速定位到对应的大版本
